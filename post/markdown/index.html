Markdown

Markdown is a lightweight markup language for creating formatted text using a plain-text editor. John Gruber and Aaron Swartz created Markdown in 2004 as a markup language that is appealing to human readers in its source code form. Markdown is widely used in blogging, instant messaging, online forums, collaborative software, documentation pages, and readme files.

Syntax Markdown

Below are examples of using Markdown, but this is far from a complete guide. A full description of the language can be found on the official website. Characters that are usually treated as special in Markdown can be escaped using a backslash. For example, the sequence “*” will output the character “*”, and will not be a sign of the beginning of the selected text. In addition, Markdown does not convert text inside “raw” XHTML blocks. Thus, you can include XHTML sections in a Markdown document by enclosing them in block-level tags beforehand.

1. Text with emphasis or logical emphasis

“* highlighting *”

“** strong highlighting **”

2. Program code

Code elements can be inside a line (inline) or multi-line blocks.

Inside the string, the code is highlighted with the symbol “`” (reverse apostrophe).

  • Sample code inside the line (inline) Hello world!

A multiline block of code is indicated by an indent of 4 spaces or one Tab.

A multi-line block of code begins below

<!doctype html>
<html>
    <head>
        <!-- Document title -->
    </head>
    <body>
        <!-- Document body -->
    </body>
</html>

The code block has ended

3. Strikethrough text

it is necessary to do ~~ one ~~ other thing

4. Lists

“*” bulleted list item

“-” another element of the unnumbered list

“+” bullet elements can be different

“1.” Numbered list item

“2.” Item #2 of the same list

“9.” Item #3 of the list — the items are numbered in order, the number at the beginning of the line does not matter

5. Headlines

Titles are created by placing a grid sign in front of the title text. The number of characters “#” corresponds to the level of the title. HTML provides 6 levels of headers.

/# The header of the first level

/### Third level header

/###### Sixth level header

  • The first two header levels also have an alternative syntax:

First level header:

/========================

Second level header:

/————————

6. Links

[Link text](http://example.com / “Optional link header”)

an alternative way to set links if there are repetitions

Somewhere among the text there is [link text][example].

Also the link repeats [example address][example].

Link to [second][foo] also [Bar][] resources.

[example]: http://example.com / “Optional link header” [foo]: http://example.net / ‘Optional link header’ [bar]: http://example.edu / (Optional link header)
